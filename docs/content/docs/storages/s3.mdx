The S3 Storage Adapter supports **AWS S3**, **Cloudflare R2**, **MinIO**, and any other S3-compatible object storage provider.

## Installation

```bash
pnpm add @better-media/adapter-storage-s3
```

## Usage

## Environment Variables

We recommend storing credentials in environment variables:

```bash title=".env"
S3_BUCKET=my-media-bucket
AWS_REGION=us-east-1
AWS_ACCESS_KEY_ID=AKIA...
AWS_SECRET_ACCESS_KEY=...
```

## Configuration Options

| Option            | Type                 | Default | Description                                                            |
| ----------------- | -------------------- | ------- | ---------------------------------------------------------------------- |
| `bucket`          | `string`             | —       | **Required.** S3 bucket name.                                          |
| `region`          | `string`             | —       | **Required.** AWS region (e.g. `"us-east-1"`, `"auto"` for R2).        |
| `accessKeyId`     | `string`             | —       | AWS access key ID.                                                     |
| `secretAccessKey` | `string`             | —       | AWS secret access key.                                                 |
| `endpoint`        | `string`             | —       | Custom endpoint URL (for R2, MinIO, etc.).                             |
| `forcePathStyle`  | `boolean`            | `false` | Use path-style URLs instead of subdomain-style. Required for MinIO.    |
| `publicUrl`       | `string \| Function` | —       | Override the public URL returned for files. Useful for CloudFront CDN. |
| `prefix`          | `string`             | `""`    | Key prefix for all uploaded objects (e.g. `"uploads/"`)                |
| `acl`             | `string`             | —       | S3 ACL for uploaded objects (e.g. `"public-read"`).                    |

## Using a CDN (CloudFront)

If your S3 bucket is fronted by CloudFront, set `publicUrl` to return the CDN URL:

```ts
new S3StorageAdapter({
  bucket: "my-bucket",
  region: "us-east-1",
  accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
  secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
  publicUrl: (key) => `https://cdn.example.com/${key}`,
});
```

## Dynamic Bucket Resolution

For multi-tenant apps, you can resolve the bucket per request:

```ts
new S3StorageAdapter({
  bucket: (key) => {
    // resolve bucket per key
    return `tenant-media`;
  },
  region: "us-east-1",
  accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
  secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
});
```
