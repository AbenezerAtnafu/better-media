---
title: Filesystem Storage
description: Store files on the local filesystem for development and testing.
---

The Filesystem Storage Adapter persists files to a directory on the local disk. It is ideal for **local development** and **testing** where you don't want to configure cloud credentials.

<Callout type="warn">
  The filesystem adapter is **not recommended for production**. Files stored locally are not replicated, not served via CDN, and not scalable across multiple server instances. Use the [S3 adapter](/docs/storages/s3) for production workloads.
</Callout>

## Installation

```bash
pnpm add @better-media/adapter-storage-filesystem
```

## Basic Usage

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";
import { filesystemStorage } from "@better-media/adapter-storage-filesystem";

export const media = createBetterMedia({
  storage: filesystemStorage({
    directory: "./uploads",
    publicUrl: "http://localhost:3000/uploads",
  }),
  // ...
});
```

## Serving Files

The adapter stores files under the `directory` path. To serve them via HTTP, configure your framework to serve the directory as static files:

<Tabs items={["Next.js", "Express"]}>
  <Tab value="Next.js">
    ```ts title="next.config.mjs"
    // Point public/* to ./uploads at dev time
    // Or use a dedicated /uploads route handler
    ```
    
    Create a catch-all route to serve uploads:

    ```ts title="app/uploads/[...path]/route.ts"
    import { NextRequest } from "next/server";
    import { readFile } from "fs/promises";
    import path from "path";

    export async function GET(req: NextRequest, { params }: { params: { path: string[] } }) {
      const filePath = path.join(process.cwd(), "uploads", ...params.path);
      const file = await readFile(filePath);
      return new Response(file);
    }
    ```
  </Tab>
  <Tab value="Express">
    ```ts title="server.ts"
    import express from "express";
    import path from "path";

    const app = express();
    app.use("/uploads", express.static(path.join(process.cwd(), "uploads")));
    ```
  </Tab>
</Tabs>

## Configuration Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `directory` | `string` | — | **Required.** Absolute or relative path to the storage directory. |
| `publicUrl` | `string` | — | **Required.** Base URL used to generate public file URLs. |
| `createDirectory` | `boolean` | `true` | Automatically create the directory if it doesn't exist. |
| `preserveFilenames` | `boolean` | `false` | Use original filenames instead of generated UUIDs. |

## Switching to S3 in Production

The filesystem adapter is designed to be a drop-in development replacement for S3. Since both implement `StorageAdapter`, switching is a one-line change:

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";

// Development
import { filesystemStorage } from "@better-media/adapter-storage-filesystem";
// Production
import { s3Storage } from "@better-media/adapter-storage-s3";

const isDev = process.env.NODE_ENV === "development";

export const media = createBetterMedia({
  storage: isDev
    ? filesystemStorage({ directory: "./uploads", publicUrl: "http://localhost:3000/uploads" })
    : s3Storage({ bucket: process.env.S3_BUCKET!, region: process.env.AWS_REGION! }),
  // ...
});
```
