---
title: Metadata Plugin
description: Automatically extract and persist technical metadata from uploaded images, videos, and audio files.
---

The Metadata Plugin automatically extracts technical metadata from uploaded files — dimensions, duration, codec, bitrate, GPS coordinates — and persists it to your database alongside the media record.

## Installation

```bash
pnpm add @better-media/plugin-metadata
```

The plugin uses [`sharp`](https://sharp.pixelplumbing.com/) for image metadata and [`ffprobe`](https://ffmpeg.org/) for video/audio. Both are **peer dependencies**:

```bash
# Sharp (for image metadata)
pnpm add sharp

# FFprobe (for video/audio metadata)
# macOS
brew install ffmpeg
# Ubuntu
apt-get install ffmpeg
```

## Basic Usage

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";
import { metadataPlugin } from "@better-media/plugin-metadata";

export const media = createBetterMedia({
  storage: /* ... */,
  database: /* ... */,
  plugins: [
    metadataPlugin({
      extractFor: ["image", "video", "audio"],
    }),
  ],
});
```

## Configuration Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `extractFor` | `MediaType[]` | `["image", "video", "audio"]` | File categories to extract metadata for. |
| `includeExif` | `boolean` | `true` | Include EXIF data for images (camera model, GPS, etc.). |
| `includeGps` | `boolean` | `false` | Include GPS coordinates if present in EXIF. Privacy consideration. |
| `mode` | `"sync" \| "background"` | `"sync"` | Run extraction inline or in a background job. |
| `onError` | `"skip" \| "fail"` | `"skip"` | What to do if metadata extraction fails. |

## Extracted Fields

### Images

| Field | Type | Description |
| --- | --- | --- |
| `width` | `number` | Width in pixels. |
| `height` | `number` | Height in pixels. |
| `format` | `string` | Image format (e.g. `"jpeg"`, `"png"`, `"webp"`). |
| `colorSpace` | `string` | Color space (e.g. `"srgb"`, `"p3"`). |
| `hasAlpha` | `boolean` | Whether the image has an alpha channel. |
| `exif` | `object` | Raw EXIF data (if `includeExif: true`). |
| `gps` | `{ lat, lng }` | GPS coordinates (if `includeGps: true`). |

### Videos

| Field | Type | Description |
| --- | --- | --- |
| `width` | `number` | Video width in pixels. |
| `height` | `number` | Video height in pixels. |
| `duration` | `number` | Duration in seconds. |
| `fps` | `number` | Frames per second. |
| `codec` | `string` | Video codec (e.g. `"h264"`, `"vp9"`). |
| `bitrate` | `number` | Bitrate in bits per second. |
| `audioCodec` | `string` | Audio codec (e.g. `"aac"`, `"opus"`). |

### Audio

| Field | Type | Description |
| --- | --- | --- |
| `duration` | `number` | Duration in seconds. |
| `bitrate` | `number` | Bitrate in bits per second. |
| `sampleRate` | `number` | Sample rate in Hz. |
| `channels` | `number` | Number of audio channels. |
| `codec` | `string` | Audio codec. |

## Accessing Metadata

After upload, metadata is available on the media record via the database:

```ts
const mediaRecord = await media.getMedia(id);

console.log(mediaRecord.metadata);
// {
//   width: 1920,
//   height: 1080,
//   format: "jpeg",
//   colorSpace: "srgb",
//   hasAlpha: false,
// }
```
