---
title: Validation Plugin
description: Validate incoming files by type, size, MIME type, and checksum before they enter the processing pipeline.
---

The Validation Plugin runs synchronous checks on incoming files as the first stage of the pipeline. If a file fails any check, the upload is rejected immediately — before any scanning or storage occurs.

## Installation

```bash
pnpm add @better-media/plugin-validation
```

## Basic Usage

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";
import { validationPlugin } from "@better-media/plugin-validation";

export const media = createBetterMedia({
  storage: /* ... */,
  database: /* ... */,
  plugins: [
    validationPlugin({
      maxSize: 10 * 1024 * 1024, // 10 MB
      allowedTypes: ["image/jpeg", "image/png", "image/webp", "video/mp4"],
    }),
  ],
});
```

## Configuration Options

| Option | Type | Default | Description |
| --- | --- | --- | --- |
| `maxSize` | `number` | `Infinity` | Maximum file size in bytes. |
| `minSize` | `number` | `0` | Minimum file size in bytes. |
| `allowedTypes` | `string[]` | `[]` (any) | Allowed MIME types. Empty array allows all. |
| `blockedTypes` | `string[]` | `[]` | MIME types that are explicitly rejected. |
| `allowedExtensions` | `string[]` | `[]` (any) | Allowed file extensions (e.g. `["jpg", "png"]`). |
| `validateMagicBytes` | `boolean` | `false` | Read the file's magic bytes to verify its true type, ignoring the declared MIME type. |
| `onReject` | `(ctx) => void` | — | Optional callback fired when a file is rejected. |

## Examples

### Image-only Upload

```ts
validationPlugin({
  maxSize: 5 * 1024 * 1024,   // 5 MB
  allowedTypes: [
    "image/jpeg",
    "image/png",
    "image/gif",
    "image/webp",
    "image/avif",
  ],
  validateMagicBytes: true,   // verify the file is actually what it claims
})
```

### Video Upload with Size Limits

```ts
validationPlugin({
  maxSize: 500 * 1024 * 1024,   // 500 MB
  minSize: 1024,                 // at least 1 KB
  allowedTypes: ["video/mp4", "video/webm", "video/quicktime"],
  allowedExtensions: ["mp4", "webm", "mov"],
})
```

### Block Dangerous Types

```ts
validationPlugin({
  blockedTypes: [
    "application/x-msdownload",
    "application/x-executable",
    "application/x-sh",
  ],
})
```

## Error Responses

When validation fails, the handler returns a structured `400 Bad Request` error:

```json
{
  "error": "VALIDATION_FAILED",
  "code": "FILE_TOO_LARGE",
  "message": "File size 15728640 exceeds the maximum allowed size of 10485760 bytes.",
  "details": {
    "receivedSize": 15728640,
    "maxSize": 10485760
  }
}
```

Possible error codes:

| Code | Description |
| --- | --- |
| `FILE_TOO_LARGE` | File exceeds `maxSize`. |
| `FILE_TOO_SMALL` | File is below `minSize`. |
| `INVALID_MIME_TYPE` | MIME type not in `allowedTypes` or in `blockedTypes`. |
| `INVALID_EXTENSION` | File extension not in `allowedExtensions`. |
| `MAGIC_BYTES_MISMATCH` | Magic bytes don't match the declared MIME type. |
