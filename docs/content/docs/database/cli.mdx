title: CLI (Schema)
description: Generate and migrate the Better Media database schema.
---

Better Media ships with a built-in CLI for managing the schema required by core features and plugins.

## Commands

### Generate

Generate incremental SQL migrations from Better Media’s central schema definition. Each run creates a **timestamped folder** under your migrations directory (e.g. `better-media-migrations/1730000000000/`) containing `migration.sql` and `snapshot.json`. The snapshot is the schema **after** that migration (so the next run only emits SQL for new changes). If the schema already matches the latest snapshot, **no new folder** is written.

```bash
npx media@latest generate
```

Each migration is written as `<migrationsDir>/<timestamp>/migration.sql` alongside `<migrationsDir>/<timestamp>/snapshot.json`. Output path is not configurable via a separate flag; set `migrationsDir` or `schemaOutput` in your config (see below). Older projects may still have a legacy root `snapshot.json`; that path is still read for planning.

Common options:

```bash
npx media@latest generate --dialect postgres
```

### Migrate (Kysely only)

Apply the Better Media schema directly to your database (available when using the built-in Kysely adapter):

```bash
npx media@latest migrate
```

Common options:

```bash
npx media@latest migrate --mode diff
```

If you’re using a different adapter (e.g. Prisma/Drizzle), use `generate` and apply the schema using your ORM’s migration tooling.

## Project config

The CLI loads a dedicated Better Media config file from your project that exports either:

- `database`: an initialized `DatabaseAdapter`, or
- `createDatabase()`: a factory returning a `DatabaseAdapter`

Default discovery order:

- `media.config.ts`
- `media.config.js`
- `better-media.config.ts`
- `better-media.config.js`

You can also pass an explicit config path:

```bash
npx media@latest generate --config ./media.config.ts
```

TypeScript config files require `ts-node` in your app.

Example config:

```ts title="media.config.ts"
import { kyselyAdapter } from "@better-media/adapter-db/kysely";
import { schema } from "better-media";
import { Kysely, SqliteDialect } from "kysely";
import Database from "better-sqlite3";

const db = new Kysely({
  dialect: new SqliteDialect({ database: new Database("dev.db") }),
});

export default {
  createDatabase: () =>
    kyselyAdapter(db, {
      config: { provider: "sqlite" },
      schema,
    }),
  dialect: "sqlite",
  // Directory for timestamped subfolders, each with migration.sql + snapshot.json (recommended).
  migrationsDir: "better-media-migrations",
  // Alternatively, omit migrationsDir and set schemaOutput to any path under the
  // desired folder — the directory is used; the .sql basename is ignored for new files.
};
```

`migrationsDir` overrides the directory implied by `schemaOutput` when both are set.

## Programmatic adapter resolution (Better Auth-style)

Better Media also exposes a single helper to resolve your adapter from an options object (Better Auth-style):

```ts title="example.ts"
import { getAdapter } from "better-media";
import config from "./media.config.js";

const adapter = await getAdapter(config);
```
