title: Installation

This guide walks you through setting up Better Media from scratch — installing packages, configuring adapters, and mounting the handler in your framework of choice.

## 1. Install the Package

Start by installing the core framework:

<Tabs items={["npm", "pnpm", "yarn", "bun"]}>
  <Tab value="npm">
    ```bash
    npm install better-media @better-media/core
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm add better-media @better-media/core
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn add better-media @better-media/core
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun add better-media @better-media/core
    ```
  </Tab>
</Tabs>

## 2. Choose a Storage Adapter

Better Media needs a storage provider to know where to persist your files. Install the adapter for your provider:

<Tabs items={["S3 / R2 / MinIO", "Filesystem (Dev)"]}>
  <Tab value="S3 / R2 / MinIO">
    ```bash
    pnpm add @better-media/adapter-storage-s3
    ```
  </Tab>
  <Tab value="Filesystem (Dev)">
    ```bash
    pnpm add @better-media/adapter-storage-filesystem
    ```
  </Tab>
</Tabs>

## 3. Choose a Database Adapter

Better Media uses a database adapter to track media metadata and job state:

```bash
pnpm add @better-media/adapter-db
```

The `@better-media/adapter-db` package includes built-in adapters for **Memory** (great for testing), **PostgreSQL via Drizzle**, and **MongoDB**.

## 4. Create Your Media Instance

Create a file at `lib/media.ts` (or `src/lib/media.ts`) and configure Better Media:

<Tabs items={["S3 + PostgreSQL", "S3 + Memory (Dev)", "Filesystem + Memory (Dev)"]}>
  <Tab value="S3 + PostgreSQL">
    ```ts title="lib/media.ts"
    import { createBetterMedia } from "better-media";
    import { s3Storage } from "@better-media/adapter-storage-s3";
    import { drizzleAdapter } from "@better-media/adapter-db/drizzle";
    import { db } from "@/db"; // your drizzle instance

    export const media = createBetterMedia({
      storage: s3Storage({
        bucket: process.env.S3_BUCKET!,
        region: process.env.AWS_REGION!,
        credentials: {
          accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
          secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
        },
      }),
      database: drizzleAdapter(db, { provider: "pg" }),
    });
    ```
  </Tab>
  <Tab value="S3 + Memory (Dev)">
    ```ts title="lib/media.ts"
    import { createBetterMedia } from "better-media";
    import { s3Storage } from "@better-media/adapter-storage-s3";
    import { memoryDatabase } from "@better-media/adapter-db";

    export const media = createBetterMedia({
      storage: s3Storage({
        bucket: "my-bucket",
        region: "us-east-1",
      }),
      database: memoryDatabase(),
    });
    ```
  </Tab>
  <Tab value="Filesystem + Memory (Dev)">
    ```ts title="lib/media.ts"
    import { createBetterMedia } from "better-media";
    import { filesystemStorage } from "@better-media/adapter-storage-filesystem";
    import { memoryDatabase } from "@better-media/adapter-db";

    export const media = createBetterMedia({
      storage: filesystemStorage({
        directory: "./uploads",
        publicUrl: "http://localhost:3000/uploads",
      }),
      database: memoryDatabase(),
    });
    ```
  </Tab>
</Tabs>

## 5. Add Plugins

Plugins extend the pipeline with additional capabilities. Install and register them:

```bash
pnpm add @better-media/plugin-validation @better-media/plugin-virus-scan
```

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";
import { s3Storage } from "@better-media/adapter-storage-s3";
import { memoryDatabase } from "@better-media/adapter-db";
import { validationPlugin } from "@better-media/plugin-validation";
import { virusScanPlugin } from "@better-media/plugin-virus-scan";

export const media = createBetterMedia({
  storage: s3Storage({ bucket: "my-bucket", region: "us-east-1" }),
  database: memoryDatabase(),
  plugins: [
    validationPlugin({
      maxSize: 10 * 1024 * 1024, // 10 MB
      allowedTypes: ["image/jpeg", "image/png", "image/webp", "video/mp4"],
    }),
    virusScanPlugin({
      scanner: "clamav",
      onThreat: "reject",
    }),
  ],
});
```

## 6. Mount the Handler

Better Media exposes a request handler that you mount as a route in your framework. All upload and retrieval operations go through this handler.

<Tabs items={["Next.js (App Router)", "Next.js (Pages Router)", "Express", "Fastify"]}>
  <Tab value="Next.js (App Router)">
    ```ts title="app/api/media/[...path]/route.ts"
    import { media } from "@/lib/media";

    export const { GET, POST, DELETE } = media.handler();
    ```
  </Tab>
  <Tab value="Next.js (Pages Router)">
    ```ts title="pages/api/media/[...path].ts"
    import { media } from "@/lib/media";

    export default media.toNextHandler();
    export const config = { api: { bodyParser: false } };
    ```
  </Tab>
  <Tab value="Express">
    ```ts title="server.ts"
    import express from "express";
    import { media } from "./lib/media";

    const app = express();
    app.use("/api/media", media.toExpressHandler());
    ```
  </Tab>
  <Tab value="Fastify">
    ```ts title="server.ts"
    import Fastify from "fastify";
    import { media } from "./lib/media";

    const app = Fastify();
    app.register(media.toFastifyPlugin(), { prefix: "/api/media" });
    ```
  </Tab>
</Tabs>

## 7. Upload a File

You can now upload files through the mounted handler using a standard `multipart/form-data` POST request, or use the Better Media client:

```ts
const formData = new FormData();
formData.append("file", file); // File object from an <input> element

const response = await fetch("/api/media/upload", {
  method: "POST",
  body: formData,
});

const result = await response.json();
// { id: "...", url: "...", metadata: { ... } }
```
