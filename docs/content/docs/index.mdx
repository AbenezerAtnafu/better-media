Better Media is a **framework-agnostic, modular media orchestration framework** for TypeScript. It provides a comprehensive pipeline for managing the entire lifecycle of media files — from intake and validation, to virus scanning, processing, and multi-destination storage.

Whether you're building a simple file upload endpoint or a distributed media processing engine with virus scanning, automated transformations, and metadata extraction, Better Media gives you the architecture to stay clean, type-safe, and scalable.

## Architecture

Better Media is built around a strict **Core / Adapter / Framework** separation:

| Layer                           | Responsibility                                                                  |
| ------------------------------- | ------------------------------------------------------------------------------- |
| **Core** (`@better-media/core`) | Defines contracts (interfaces, types). No implementations. The source of truth. |
| **Adapters**                    | Implement infrastructure integrations (S3, Postgres, Redis).                    |
| **Framework** (`better-media`)  | Orchestrates the pipeline — wires adapters and plugins together.                |

## Official Packages

| Package                                    | Description                                                    |
| ------------------------------------------ | -------------------------------------------------------------- |
| `better-media`                             | Framework entry point. Creates and manages the media pipeline. |
| `@better-media/core`                       | Shared interfaces and types.                                   |
| `@better-media/adapter-storage-s3`         | S3-compatible storage (AWS, MinIO, Cloudflare R2).             |
| `@better-media/adapter-storage-filesystem` | Local filesystem storage for development.                      |
| `@better-media/adapter-db`                 | Database adapters (Memory, PostgreSQL, MongoDB).               |
| `@better-media/plugin-validation`          | File validation (type, size, MIME, checksum).                  |
| `@better-media/plugin-virus-scan`          | Virus scanning via ClamAV or VirusTotal.                       |
| `@better-media/plugin-metadata`            | Metadata extraction for images, video, and audio.              |
| `@better-media/plugin-media-processing`    | Image resizing and thumbnail generation via Sharp.             |

## Quick Example

```ts title="lib/media.ts"
import { createBetterMedia } from "better-media";
import { s3Storage } from "@better-media/adapter-storage-s3";
import { memoryDatabase } from "@better-media/adapter-db";
import { validationPlugin } from "@better-media/plugin-validation";
import { virusScanPlugin } from "@better-media/plugin-virus-scan";

export const media = createBetterMedia({
  storage: s3Storage({
    bucket: "my-media-bucket",
    region: "us-east-1",
  }),
  database: memoryDatabase(),
  plugins: [
    validationPlugin({
      maxSize: 10 * 1024 * 1024, // 10MB
      allowedTypes: ["image/jpeg", "image/png", "image/webp"],
    }),
    virusScanPlugin({
      scanner: "clamav",
      onThreat: "reject",
    }),
  ],
});
```

## FAQ

**Is Better Media production-ready?**
Better Media is actively developed. The core interfaces are stable, but some adapters and plugins are still evolving. We recommend pinning to a specific version in production.

**Does it work with Next.js App Router?**
Yes. Better Media is framework-agnostic and works with Next.js App Router, Pages Router, Express, Fastify, Hono, and any other Node.js-compatible framework.

**Can I use my own database ORM?**
Yes. The `DatabaseAdapter` interface is simple enough to implement for any ORM or driver in a few dozen lines of code.

**Does it support streaming uploads?**
Background job-based processing (async mode) naturally supports streaming. Direct sync mode depends on the framework's request handling.
